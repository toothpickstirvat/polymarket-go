# Polymarket Go 底层客户端接口调研报告

**文档版本**: v1.0
**更新日期**: 2025-11-25
**调研目标**: 全面了解所有 Polymarket Go 底层客户端的能力，为高层封装设计提供依据

---

## 执行摘要

本文档详细调研了 Polymarket Go 生态系统中的 5 个核心底层客户端：
- **Gamma Client**: 市场元数据和事件信息
- **Data Client**: 用户数据、持仓和交易历史
- **Realtime Client**: WebSocket 实时数据流
- **CLOB Client**: 订单簿交易和市场数据
- **Contracts Client**: 以太坊合约交互

这些客户端共同提供了完整的 Polymarket 交易生态能力，覆盖从市场发现、数据查询、实时监控到交易执行的全流程。

---

## 1. Gamma Client (v0.2.1)

### 1.1 核心定位
**市场元数据和事件信息查询中心**

Gamma Client 是获取 Polymarket 市场结构化信息的主要入口，提供市场、事件、系列、标签等静态和准静态数据。

### 1.2 主要功能模块

#### 1.2.1 Markets (市场)
- **接口数量**: 4个
- **核心能力**:
  - 多条件过滤查询市场列表（流动性、交易量、时间范围等）
  - 按 ID 或 Slug 获取单个市场详情
  - 获取市场关联的标签

- **关键数据字段**:
  ```go
  Market {
    ID, Question, ConditionID, Slug
    Liquidity, LiquidityNum, Volume, VolumeNum
    LastTradePrice, BestBid, BestAsk, Spread
    OutcomePrices, Outcomes
    Active, Closed, Featured
    Volume24hr, Volume1wk, Volume1mo, Volume1yr
    OneDayPriceChange, OneWeekPriceChange
    EnableOrderBook, AcceptingOrders
    MakerBaseFee, TakerBaseFee
    // 100+ 字段
  }
  ```

#### 1.2.2 Events (事件)
- **接口数量**: 4个
- **核心能力**:
  - 查询事件列表（支持标签过滤、时间过滤、精选筛选）
  - 按 ID 或 Slug 获取事件详情
  - 获取事件关联的市场、系列、标签

- **应用场景**: 主题化市场分组、体育赛事、选举专题等

#### 1.2.3 Series (系列)
- **接口数量**: 2个
- **核心能力**: 系列化市场的管理和查询（如季度性市场、重复性事件）

#### 1.2.4 Tags (标签)
- **接口数量**: 7个
- **核心能力**:
  - 标签列表和详情查询
  - 相关标签发现（相关性推荐）
  - 标签关系图谱

#### 1.2.5 Search (搜索)
- **接口数量**: 1个
- **核心能力**: 统一搜索入口，支持搜索市场、事件、用户个人资料

#### 1.2.6 Sports (体育)
- **接口数量**: 2个
- **核心能力**:
  - 团队信息查询
  - 体育元数据（联赛、解决源、排序信息）

### 1.3 特色功能
- **图像优化**: 所有图像都提供优化版本，包含源和优化图像的大小对比
- **时间灵活处理**: NormalizedTime 支持多种时间格式自动解析
- **丰富的过滤器**: 所有列表查询都支持强大的多条件过滤和排序

### 1.4 对高层封装的价值
✅ **市场发现**: 策略需要根据流动性、交易量等指标筛选优质市场
✅ **市场元数据**: 提供决策所需的市场基础信息
✅ **关联分析**: 通过标签和系列发现相关市场机会

## 2. Data Client (v0.1.1)

### 2.1 核心定位
**用户数据和交易历史中心**

Data Client 专注于用户视角的数据，提供持仓管理、交易历史、活动追踪和市场统计数据。

### 2.2 主要功能模块

#### 2.2.1 Positions (持仓管理)
**核心接口**:
- `GetPositions`: 获取当前持仓
- `GetClosedPositions`: 获取已平仓头寸
- `GetPositionsValue`: 获取持仓总价值

**Position 数据结构**:
```go
Position {
  Size               // 持仓规模
  AvgPrice           // 平均价格
  InitialValue       // 初始价值
  CurrentValue       // 当前价值
  CashPnl            // 现金盈亏
  PercentPnl         // 百分比盈亏
  TotalBought        // 总购买量
  RealizedPnl        // 已实现盈亏
  PercentRealizedPnl // 已实现百分比盈亏
  CurPrice           // 当前价格
  Redeemable         // 是否可赎回
  Mergeable          // 是否可合并
}
```

**过滤能力**:
- 按市场或事件过滤
- 按规模阈值过滤
- 按可赎回/可合并状态过滤
- 支持分页和多维度排序

#### 2.2.2 Trades (交易记录)
**核心接口**:
- `GetTrades`: 获取交易历史
- `GetTradedMarketsCount`: 获取交易过的市场总数

**Trade 数据结构**:
```go
Trade {
  Side         // BUY/SELL
  Size         // 交易数量
  Price        // 交易价格
  Timestamp    // 时间戳
  ConditionId  // 市场ID
  Outcome      // 结果
  TransactionHash // 交易哈希
  // 用户信息
  Name, Pseudonym, ProfileImage
}
```

**过滤能力**:
- 按用户、市场、事件过滤
- 按交易方向过滤 (BUY/SELL)
- 按金额或代币数量过滤
- 仅 Taker 或包含 Maker

#### 2.2.3 Activity (活动追踪)
**核心接口**:
- `GetActivity`: 获取用户链上活动

**活动类型**:
- `TRADE`: 交易
- `SPLIT`: 拆分
- `MERGE`: 合并
- `REDEEM`: 赎回
- `REWARD`: 奖励
- `CONVERSION`: 转换

**应用场景**: 完整的用户行为追踪，用于策略分析和税务报告

#### 2.2.4 Holders (持有人)
**核心接口**:
- `GetHolders`: 获取市场代币的顶级持有人

**应用场景**: 大户追踪、市场情绪分析

#### 2.2.5 Market Metrics (市场指标)
**核心接口**:
- `GetOpenInterest`: 获取未平仓头寸
- `GetLiveVolume`: 获取实时交易量

**应用场景**: 市场活跃度评估、流动性监控

### 2.3 数据精度
- **小数处理**: 使用 `shopspring/decimal` 库确保精确计算
- **时间精度**: 支持毫秒级时间戳

### 2.4 对高层封装的价值
✅ **账户资产概览**: 提供完整的持仓和价值数据
✅ **收益计算**: PnL、已实现盈亏等核心财务指标
✅ **交易统计**: 胜率、盈亏比计算的数据基础
✅ **活动追踪**: 完整的用户行为数据

## 3. Realtime Client (v0.3.0)

### 3.1 核心定位
**实时数据流中心**

Realtime Client 通过 WebSocket 提供低延迟的实时数据流，支持价格更新、交易流、订单簿变化等。

### 3.2 主要功能模块

#### 3.2.1 连接管理
- **自动重连**: 支持指数退避的自动重连机制
- **心跳机制**: Ping 循环保持连接活跃
- **订阅恢复**: 重连后自动恢复所有订阅

#### 3.2.2 订阅主题

| 主题 | 消息类型 | 用途 |
|------|---------|------|
| `activity` | `trades`, `orders_matched` | 交易活动流 |
| `comments` | `comment_created/removed`, `reaction_created/removed` | 社区互动 |
| `rfq` | `request_*`, `quote_*` | RFQ 报价系统 |
| `crypto_prices` | `update` | 加密货币价格 (Binance) |
| `crypto_prices_chainlink` | `update` | Chainlink 价格预言机 |
| `equity_prices` | `update` | 股票价格 |
| `clob_user` | `order`, `trade` | 用户订单和交易 |
| `clob_market` | `price_change`, `agg_orderbook`, `last_trade_price`, `tick_size_change`, `market_created`, `market_resolved` | CLOB 市场数据 |

#### 3.2.3 类型安全订阅

**Realtime Typed Subscription Handler** 提供强类型的回调接口：

```go
// 交易活动
SubscribeToActivityTrades(callback func(trade Trade) error, filter *ActivityFilter)

// CLOB 价格变化
SubscribeToCLOBMarketPriceChanges(callback func(changes PriceChanges) error, filter *CLOBMarketFilter)

// 加密货币价格
SubscribeToCryptoPrices(callback func(price CryptoPrice) error, filter *CryptoPriceFilter)

// 订单簿
SubscribeToCLOBMarketAggOrderbook(callback func(orderbook AggOrderbook) error, filter *CLOBMarketFilter)
```

#### 3.2.4 过滤器系统

**ActivityFilter**: 按市场或资产过滤交易
**CommentFilter**: 按事件或系列过滤评论
**CryptoPriceFilter**: 指定加密货币符号 (每连接限1个)
**EquityPriceFilter**: 指定股票代码 (每连接限1个)
**CLOBMarketFilter**: 指定代币 ID 列表 (支持多个)

#### 3.2.5 数据结构

**Trade**: 完整的交易详情，包含价格、规模、方向、时间戳、用户信息
**PriceChanges**: 订单簿价格变化事件
**AggOrderbook**: 聚合订单簿快照
**CLOBOrder**: CLOB 订单更新
**CLOBTrade**: CLOB 交易匹配

### 3.3 特色功能
- **多订阅支持**: 大部分主题支持同一连接的多订阅
- **消息路由**: 自动路由消息到注册的类型化处理器
- **断线重连**: 自动恢复连接和订阅

### 3.4 对高层封装的价值
✅ **实时价格监控**: 策略执行需要实时价格数据
✅ **订单簿深度**: 大单执行和滑点估算
✅ **交易流分析**: 市场情绪和动量判断
✅ **事件驱动策略**: 基于实时事件触发交易逻辑

## 4. CLOB Client (订单簿客户端)

### 4.1 核心定位
**订单簿交易和市场数据中心**

CLOB Client 是交易执行的核心，提供完整的订单生命周期管理、市场数据查询和交易历史。

### 4.2 主要功能模块

#### 4.2.1 身份验证系统

**L1 认证** (签名者):
```go
CreateApiKey()        // 创建新 API Key
DeriveApiKey()        // 推导 API Key (确定性)
CreateOrDeriveApiKey() // 自动选择
```

**L2 认证** (API Key):
- API Key
- Secret
- Passphrase

**认证流程**: 签名者签署 EIP-712 消息 → 获取 API 凭证 → 使用 API Key 调用接口

#### 4.2.2 订单管理

**限价单**:
```go
CreateOrder()          // 创建并签名订单
PostOrder()            // 发送订单到匹配引擎
CreateAndPostOrder()   // 一步完成
```

**市价单**:
```go
CreateMarketOrder()           // 创建市价单 (自动计算价格)
CreateAndPostMarketOrder()    // 一步完成
CalculateMarketPrice()        // 根据订单簿计算市价
```

**订单类型**:
- `GTC` (Good-Til-Cancelled): 持续有效直到手动取消
- `GTD` (Good-Til-Date): 到期自动取消
- `FOK` (Fill-Or-Kill): 全部成交或全部取消
- `FAK` (Fill-And-Kill): 部分成交后取消剩余

**订单查询**:
```go
GetOpenOrders()  // 获取开放订单 (支持分页和过滤)
GetOrder()       // 获取单个订单详情
```

**订单取消**:
```go
CancelOrder()            // 取消单个订单
CancelOrders()           // 批量取消
CancelOrdersByMarket()   // 按市场取消
CancelAll()              // 取消所有订单
```

#### 4.2.3 市场数据

**价格接口**:
```go
GetMidpoint()        // 中点价格
GetMarketPrice()     // 单向价格 (买/卖)
GetMarketPrices()    // 批量价格
GetSpreads()         // 买卖价差
GetLastTradePrice()  // 最后交易价格
```

**订单簿**:
```go
GetOrderBook()   // 单个订单簿
GetOrderBooks()  // 批量订单簿
```

**OrderBookSummary 结构**:
```go
{
  Bids []PriceLevel    // 买单 (价格从高到低)
  Asks []PriceLevel    // 卖单 (价格从低到高)
  MinOrderSize         // 最小订单规模
  TickSize             // 最小价格单位
  Hash                 // 订单簿哈希 (版本控制)
  NegRisk              // 是否为负风险市场
}
```

**市场信息**:
```go
GetMarkets()     // 市场列表
GetMarket()      // 单个市场详情
GetTickSize()    // 最小价格单位
GetNegRisk()     // 负风险标志
```

**价格历史**:
```go
GetPriceHistory()  // K线数据
```
- 支持间隔: 1m, 1h, 6h, 1d, 1w
- 支持自定义精度 (Fidelity)

#### 4.2.4 交易历史

```go
GetTrades()  // 获取交易记录
```
- 支持按用户、市场、时间过滤
- 包含订单匹配详情
- 返回交易状态 (MATCHED, MINED, CONFIRMED)

#### 4.2.5 余额和授权

```go
GetBalanceAllowance()    // 获取余额和授权额度
UpdateBalanceAllowance() // 更新授权额度
```
- **COLLATERAL**: USDC 余额
- **CONDITIONAL**: 条件代币余额

#### 4.2.6 通知系统

```go
GetNotifications()   // 获取通知
DropNotifications()  // 标记已读 (删除)
```

### 4.3 特色功能

**精确计算**:
- 使用 `shopspring/decimal` 库
- 自动 Tick Size 对齐
- 市价单的特殊精度规则

**代理钱包支持**:
- 独立的签名者和资金地址
- Gnosis Safe 兼容

**订单安全**:
- GTD 订单过期安全阈值 (默认 60 秒)
- 防止过期订单被匹配

**辅助计算**:
```go
CalculateSharesGivenAmountAndPrice()  // 金额 → 股份
CalculateAmountGivenSharesAndPrice()  // 股份 → 金额
CalculatePriceGivenAmountAndShares()  // 金额 + 股份 → 价格
```

### 4.4 对高层封装的价值
✅ **交易执行**: 完整的订单生命周期管理
✅ **价格发现**: 实时市场价格和订单簿深度
✅ **交易历史**: 详细的交易记录用于绩效分析
✅ **余额管理**: 资金和代币余额追踪

## 5. Contracts Client (合约交互)

### 5.1 核心定位
**以太坊合约交互和链上操作**

Contracts Client 提供与 Polymarket 智能合约的交互能力，包括代币拆分/合并、赎回和 Safe 钱包管理。

### 5.2 主要功能模块

#### 5.2.1 Trading 启用

```go
EnableTrading()         // 一键启用所有交易授权
EnableTradingForSafe()  // Safe 钱包版本
EnableTradingForEOA()   // EOA 钱包版本
```

**授权内容**:
- USDC → Exchange
- USDC → NegRiskAdapter
- USDC → NegRiskExchange
- CTF → Exchange (setApprovalForAll)
- CTF → NegRiskAdapter (setApprovalForAll)
- CTF → NegRiskExchange (setApprovalForAll)

**智能检测**: 仅授权尚未授权的合约，避免重复交易

#### 5.2.2 Position 操作

**Split (拆分)**:
```go
Split(ctx, conditionId, partition, amount)  // 将 USDC 拆分为条件代币
SplitPositionForSafe()   // Safe 钱包版本
SplitPositionForEOA()    // EOA 版本
```

**Merge (合并)**:
```go
Merge(ctx, conditionId, partition, amount)  // 将条件代币合并回 USDC
MergePositionsForSafe()
MergePositionsForEOA()
```

**Redeem (赎回)**:
```go
Redeem(ctx, conditionId, indexSets)         // 赎回已结算市场的代币
RedeemPositionsForSafe()
RedeemPositionsForEOA()
```

**RedeemNegRisk (负风险赎回)**:
```go
RedeemNegRisk(ctx, conditionId, amounts)    // 负风险市场的赎回
RedeemPositionsNegRiskForSafe()
RedeemPositionsNegRiskForEOA()
```

#### 5.2.3 Safe 钱包管理

**部署 Safe**:
```go
DeploySafe()  // 部署 Gnosis Safe 代理钱包
```

**Safe 交易执行**:
```go
ExecuteTransactionBySafeAndSingleSigner()  // 单签名者执行 Safe 交易
EstimateSafeTxGas()                       // 估算 Safe 交易 gas
```

**Safe 地址计算**:
```go
GetSafeAddress(eoa)  // 根据 EOA 地址计算 Safe 代理地址
```

#### 5.2.4 余额和授权检查

```go
CheckBalanceAndAllowance(ctx, address)  // 检查余额和所有授权状态
PrintBalanceAndAllowance(ctx, address)  // 打印友好格式的余额信息
```

**返回信息**:
- USDC 余额
- 各合约的 USDC 授权额度
- CTF 合约的 setApprovalForAll 状态

#### 5.2.5 合约访问器

```go
GetCollateral()       // USDC 合约实例
GetConditionalTokens() // CTF 合约实例
GetExchange()         // Exchange 合约实例
GetNegRiskAdapter()   // NegRisk Adapter 实例
// ... 其他合约
```

### 5.3 特色功能

**签名类型支持**:
- `SignatureTypeEOA`: 外部账户
- `SignatureTypePolyGnosisSafe`: Polymarket 代理钱包

**Gas 估算**:
- 自动估算 Safe 交易的 safeTxGas
- 包含 Safe 执行开销和缓冲

**交易构建器**:
- EIP-712 类型化数据签名
- Safe 交易类型化数据
- CLOB 认证类型化数据

### 5.4 对高层封装的价值
✅ **链上操作**: 提供拆分、合并、赎回等核心操作
✅ **授权管理**: 自动化交易前的授权流程
✅ **Safe 集成**: 支持代理钱包的安全交易
✅ **余额追踪**: 链上余额和授权状态查询

## 7. 数据流向分析

### 7.1 市场发现流程
```
Gamma Client (搜索/过滤市场)
  ↓
获取市场元数据 (流动性、交易量、价格)
  ↓
CLOB Client (查询实时价格和订单簿)
  ↓
Data Client (查询未平仓头寸和交易量)
```

### 7.2 交易执行流程
```
CLOB Client (创建订单)
  ↓
Contracts Client (确保交易授权已启用)
  ↓
CLOB Client (发送订单到匹配引擎)
  ↓
Realtime Client (监听订单状态和交易匹配)
  ↓
Data Client (更新持仓和交易历史)
```

### 7.3 持仓监控流程
```
Data Client (查询当前持仓)
  ↓
Gamma Client (获取市场元数据)
  ↓
CLOB Client (查询当前市场价格)
  ↓
Realtime Client (订阅实时价格更新)
  ↓
计算持仓价值和 PnL
```

### 7.4 赎回流程
```
Data Client (查询可赎回持仓)
  ↓
Gamma Client (检查市场是否已结算)
  ↓
Contracts Client (执行 Redeem 操作)
  ↓
Data Client (验证持仓已清零)
```

## 8. 核心数据类型汇总

### 8.1 市场相关
- **Market** (Gamma): 100+ 字段的完整市场信息
- **Event** (Gamma): 事件分组和元数据
- **Token** (Gamma/CLOB): 代币信息（ID、结果、价格）

### 8.2 持仓相关
- **Position** (Data): 当前持仓（规模、均价、PnL）
- **ClosedPosition** (Data): 已平仓头寸（均价、已实现 PnL）
- **UserValue** (Data): 持仓总价值

### 8.3 交易相关
- **Trade** (Data): 链上交易记录
- **CLOBTrade** (CLOB): CLOB 交易匹配
- **Activity** (Data): 链上活动（TRADE/SPLIT/MERGE/REDEEM）

### 8.4 订单相关
- **UserOrder** (CLOB): 用户订单参数
- **SignedOrder** (CLOB): 已签名订单
- **OpenOrder** (CLOB): 开放订单状态
- **OrderResponse** (CLOB): 订单响应

### 8.5 实时数据
- **Message** (Realtime): WebSocket 消息
- **PriceChanges** (Realtime): 价格变化事件
- **AggOrderbook** (Realtime): 聚合订单簿
- **CryptoPrice** (Realtime): 加密货币价格
- **EquityPrice** (Realtime): 股票价格

### 8.6 市场数据
- **OrderBookSummary** (CLOB): 订单簿快照
- **PriceResponse** (CLOB): 价格响应
- **MidpointResponse** (CLOB): 中点价格
- **LastTradePrice** (CLOB): 最后交易价格

## 9. 限制和注意事项

### 9.1 Realtime Client
- **单符号限制**: `crypto_prices`, `crypto_prices_chainlink`, `equity_prices` 每连接限 1 个符号
- **解决方案**: 多符号追踪需要为每个符号创建独立客户端连接

### 9.2 Data Client
- **分页限制**:
  - Positions: 最大 500 条/请求
  - Trades: 最大 10000 条/请求
  - Activity: 最大 500 条/请求

### 9.3 CLOB Client
- **Tick Size 对齐**: 所有价格必须是 tickSize 的倍数
- **订单过期**: GTD 订单需要设置合理的过期时间（建议 > 60 秒）
- **API 限流**: 需要实现合理的请求频率控制

### 9.4 Contracts Client
- **Gas 费用**: 所有链上操作需要 MATIC 作为 Gas
- **交易确认**: Safe 交易需要等待链上确认
- **授权检查**: 交易前需要确保所有必要的授权已完成

## 10. 推荐的集成模式

### 10.1 单一客户端模式（当前实现）
```go
client := polymarket.NewClient(ethclient, options...)
gammaClient := client.GammaClient()
dataClient := client.DataClient()
clobClient := client.ClobClient()
// ...
```

**优点**: 统一管理，共享配置
**缺点**: 底层客户端直接暴露

### 10.2 高层封装模式（推荐）
```go
client := polymarket.NewClient(ethclient, options...)

// 高层功能模块
portfolio := client.Portfolio()        // 持仓管理
analytics := client.Analytics()        // 分析统计
strategy := client.Strategy()          // 策略支持
riskManager := client.RiskManager()    // 风险控制
```

**优点**: 面向用户需求，隐藏复杂性
**缺点**: 需要额外的封装层
